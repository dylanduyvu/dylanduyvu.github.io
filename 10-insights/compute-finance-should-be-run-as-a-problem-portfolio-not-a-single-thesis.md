# Compute finance should be run as a problem portfolio, not a single thesis

## Claim

If compute-finance is the deliberate search space, then each startup thesis should be treated as a probe inside a problem portfolio. Novation dying is one probe down, SLA verification is a live probe, and the right operating mode is cheapest-fatal-first testing across multiple candidate problems rather than over-rotating onto the next available idea.

## Why It Matters

This turns failed theses into compounding domain knowledge instead of sunk cost. The relationships, stack map, supply dynamics, and lender/insurance context remain useful even when a specific wedge dies. It also reduces emotional whiplash: if SLA fails, the process moves to the next mapped probe instead of restarting from zero.

## Evidence

- 2026-06-30: Novation was shelved for the current market, but the underlying compute-finance context still compounds.
- 2026-06-29: USD.AI volunteered SLA/uptime verification as a lender pain tied to pricing more aggressively, making it a strong immediate probe.
- 2026-06-30: The GPU KBB/naked-data thesis was demoted, not killed, pending buyer-side WTP tests.
- 2026-06-30: The current candidate portfolio already includes SLA/performance verification, naked valuation/KBB demand-side testing, bare compute-contract underwriting, and a novation tripwire.

## Implications

- Hold SLA as a probe, not the new foreground, until it replicates across Barkr and other lenders/insurers/offtakers.
- Maintain an explicit compute-finance problem inventory and test it in sequence by cheapest fatal uncertainty.
- Treat Thomas/Barkr and Bernie/American Compute as async follow-up tests because they can validate whether USD.AI's SLA pain appears from adjacent positions in the stack.
- Use emails for two jobs: validate the active probe and harvest volunteered pain to expand the inventory. No upcoming Barkr call is scheduled.
- Keep tripwires for shelved ideas instead of deleting them.

## Counterpoints / Uncertainty

- Space-level conviction can become a license to linger too long in a domain if no problem validates.
- A portfolio can become vague if each candidate does not have a clear kill criterion.
- SLA could deserve foreground quickly if two or three more qualified sources independently name the same pain and budget.
