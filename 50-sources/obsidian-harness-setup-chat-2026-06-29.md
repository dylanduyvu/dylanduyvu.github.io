# Obsidian harness setup chat 2026-06-29

## Context

Dylan designed a local-first Obsidian + Git note harness with Codex/Claude as the capture and curation layer. The design decisions came from an interview-style setup conversation.

## Raw Context

The raw conversation currently lives in this Codex thread. Future rich sources should store the raw context directly in this vault when practical.

## Summary

The vault should be insight-driven, full-fidelity, local-first, and committed after every capture/update. Agents should search before creating new notes, preserve historical claim titles when beliefs change, and maintain sparse hubs for projects, domains, people, and orgs.

## Promoted Insights

- Insight notes should use claim titles rather than topic labels.
- Search-before-create is a core dedupe rule.
- Historical claims should be preserved and marked contested/weakened/superseded instead of renamed away.
- Git history is the capture chronology in v0.

## Open Questions

- When should durable agent memories start mirroring into the vault?
- When should remote encrypted backup or Git LFS be introduced?
